# Weekly ALCO Briefing — Design Spec

**Date:** 2026-05-30
**Status:** Approved design (pre-implementation-plan)
**Author:** Adnan + Claude (brainstorming session)
**Repos touched:** YieldScope (display), EconDelta (`~/Projects/clauding-lab/econdelta`, generation), Supabase (storage)

---

## 1. Problem

YieldScope's **Briefings** page (`src/pages/Intelligence.tsx`, nav key `INT`, route `/intelligence`)
is a fully-designed but non-functional mockup:

- The "drafted by Claude" weekly read is a hardcoded string (`FX.intel.weekly` in `src/data/fixtures.ts:244`).
- The anomaly list is fixtures + inline literals; the relative ages are a positional array, not real.
- Every button (Regenerate, Edit tone, Read history, Share, Generate PDF) has **no handler**.
- The whole page is honestly tagged with `DemoBadge`.

We want this surface to become **real**: a genuine weekly ALCO read generated by Claude on the VPS
from live data, with real anomalies, prior-week history, and continuity ("persistent memory").

## 2. Goals / Non-Goals

**Goals**
1. A weekly briefing (headline + prose) generated once a week, Monday morning, by Claude Opus 4.8 (1M) at `xhigh` effort, running on the ExonVPS Dhaka box.
2. A **real anomaly list** (hybrid: numbers computed deterministically in Python, curated + explained by Claude).
3. **History** — read prior weeks' briefings in the app.
4. **Persistent memory** — the brief follows through on its own prior calls week to week (carried-forward "open threads").
5. **Honesty preserved** — `DemoBadge` hides only when a real briefing exists; a freshness gate prevents publishing a confident read on stale data.

**Non-Goals (explicitly deferred)**
- ❌ Regenerate / Edit-tone controls — **removed**.
- ❌ Working PDF export ("Generate PDF" / "Export brief") — leave mocked or strip.
- ❌ The 7-week event timeline — leave mocked.
- ❌ The ALCO decision log table — leave mocked.
- ❌ Agentic Claude with live DB tools — rejected; numbers stay in Python's hands.

## 3. Locked Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Anomaly detection | **Hybrid** — Python computes candidates, Claude curates + explains | Reproducible numbers; judgment from Claude; fits honesty governance |
| Run host | **ExonVPS Dhaka** (`adnan-local@103.187.23.22`) | EconDelta + `claude` CLI + OAuth token + systemd-timer infra already live there; data is local |
| Build scope | **Briefing + anomalies + history only** | Smallest shippable slice; defer PDF/timeline/decisions |
| Persistent memory | **Prior briefings + carried-forward open threads** | Continuity like a desk economist tracking its own calls |
| Freshness gate behavior | **Tiered** — skip on core-series staleness; generate-with-banner on peripheral | A confident stale read is worse than an honest gap; partial staleness shouldn't block the whole brief |
| Schedule | **Monday ~07:00 BDT** (`OnCalendar=Mon 01:00 UTC`) | True "Monday morning"; uses Sunday's confirmed-fresh aggregate. Trivially shiftable. |
| Model / effort | `opus[1m]` / `xhigh` | User requirement (latest Opus 4.8 + 1M context window) |
| Generation approach | **Approach A** — deterministic Python harness, Claude as writer | Mirrors the existing `opus_review.py` daily pattern; lowest risk, most reuse |

## 4. Architecture & Data Flow

```
  ExonVPS Dhaka — new systemd timer, Mon ~07:00 BDT
  ┌─────────────────────────────────────────────────────────┐
  │ briefing_weekly.py  (EconDelta)                           │
  │  1. FRESHNESS GATE — confirm core series actually         │
  │     re-parsed (run_logs + parse provenance, not as_of)    │
  │  2. GATHER  ← Supabase: metric_history (per metric_defs),  │
  │              last 8 briefings, last open_threads          │
  │  3. COMPUTE anomaly candidates in Python (per-frequency:   │
  │     z-score vs 30D mean, w/w Δ, threshold breach)          │
  │  4. CALL run_max(model="opus[1m]", effort="xhigh")         │
  │     prompt = digest + candidates + history + threads       │
  │     → JSON {title, body, featured_anomalies[], threads[]}  │
  │  5. VALIDATE json (schema; candidate_ids must exist)       │
  │     → assemble full rows → UPSERT `briefings` (PK week_of) │
  │     → write run_logs(source='briefing', cost/tokens)       │
  └─────────────────────────────────────────────────────────┘
                              │ service-role write
                              ▼
        Supabase ── `briefings` table (anon-read RLS) ──┐
                                                         │ client-side PostgREST
                              ┌──────────────────────────┘
                              ▼
  YieldScope (static PWA, GitHub Pages)
  ┌─────────────────────────────────────────────────────────┐
  │ useBriefing() → Intelligence.tsx ("Briefings")            │
  │  • latest row → headline + body + real anomaly sidebar    │
  │  • history list → "Read history" opens prior weeks        │
  │  • DemoBadge null-gated; honest "data as of" banner       │
  │  • Regenerate / Edit-tone REMOVED                         │
  └─────────────────────────────────────────────────────────┘
```

**Reuse map (do not reinvent):**
- Claude invocation → `claude_max/max_client.py::run_max` (pass `model`, `effort`, `timeout_s`).
- Gather/review pattern → `utils/opus_review.py` (`load_history`, `review_data`, `archive_latest`).
- Anon-read exposure → model the new RLS on `db/migrations/0005_metric_history_anon_read.sql`.
- Frontend data read → mirror `src/hooks/useLiquidity.ts` (client-side PostgREST, anon key).
- Badge gating → mirror CorridorViz null-gating in `src/pages/Liquidity.tsx`.

## 5. Data Model — `briefings` table

New EconDelta migration `db/migrations/0008_briefings.sql` (number confirmed: latest is 0007).

| column | type | purpose |
|---|---|---|
| `week_of` | `date` PRIMARY KEY | Monday's date — idempotency key |
| `generated_at` | `timestamptz` default now() | when the job ran |
| `title` | `text` not null | headline |
| `body` | `text` not null | weekly-read prose |
| `featured_anomalies` | `jsonb` not null default `[]` | assembled by Python: `{candidate_id,label,stat,value,severity,metric_id,why}` |
| `open_threads` | `jsonb` not null default `[]` | persistent memory: `{id,thread,status,since_week,note,last_updated_week}` |
| `data_as_of` | `date` not null | freshness stamp for the honesty banner |
| `stale_series` | `text[]` default `{}` | peripheral series flagged stale this run (drives the banner) |
| `model` | `text` not null | provenance, e.g. `opus[1m]` |
| `effort` | `text` not null | provenance, e.g. `xhigh` |
| `total_cost_usd` | `numeric` | from `run_max` telemetry |
| `inputs_hash` | `text` | optional — detect "data unchanged since last week" |

**RLS:** enable row level security; `anon` gets `SELECT` only; writes are service-role
(same shape as migration 0005). The PWA reads with the existing anon key already in
`.env.local` (`VITE_SUPABASE_URL` / `VITE_SUPABASE_ANON_KEY`).

**Memory = this table.** Each Monday the job reads the last ~8 rows + the most recent
`open_threads`, injects them into the prompt, and stores Claude's `updated_threads` back.
There is no long-lived Claude session — see §7.

## 6. Generation Flow & Prompt Contract

`briefing_weekly.py` runs the five steps above. The contract:

**Inputs Python feeds Claude (the digest):**
- Built from `metric_definitions` so it auto-covers exactly the metrics the app pages show.
- Per metric: recent series + computed stats (latest, 30D mean, σ, w/w Δ), respecting each
  metric's frequency (daily vs the monthly series from migrations 0006/0007).
- The **pre-computed anomaly candidates**, each with a stable `candidate_id` and its real numbers.
- The **last 8 briefings** (title + body + threads) and the most recent `open_threads`.
- Digest is *shaped*, not a raw row dump: full series for the headline ALCO metrics that drive
  anomalies; compact summaries for the long tail. (1M context can hold more; shaping favors
  reliability + cost. Widenable.)

**Output Claude must return (strict JSON, schema-validated):**
```json
{
  "title": "…",
  "body":  "…weekly read…",
  "featured_anomalies": [ { "candidate_id": "X-LIQ-7", "why": "…ALCO relevance, one line…" } ],
  "updated_threads":    [ { "id": "t-reserves-imf", "thread": "Reserves vs IMF floor",
                            "status": "open", "since_week": "2026-W20", "note": "…" } ]
}
```

**Integrity guarantee:** `featured_anomalies[].candidate_id` MUST reference a candidate Python
produced this run. The validator rejects any unknown id. **Claude selects + explains; it cannot
invent a number.** Python then assembles the stored anomaly objects (Python's numbers + Claude's
`why`), so the PWA receives complete, trustworthy objects.

**`run_max` call:** `--tools "" --no-session-persistence --strict-mcp-config
--permission-mode bypassPermissions --output-format json` (all already baked into `run_max`).
Pass `model="opus[1m]"`, `effort="xhigh"`, `timeout_s` generous (e.g. 1800).

## 7. Persistent Memory — the honest mechanic

The `claude -p` call remembers nothing between runs (`--no-session-persistence`). Persistence is
**DB-backed and Python-driven**: prior briefings + `open_threads` are read from Supabase and
injected into the prompt; Claude returns updated threads; Python stores them. Functionally this is
a desk economist following its own calls ("reserves vs IMF floor — still open, 3rd week"), but the
memory lives in the `briefings` table, not a Claude session. This is deliberate: it survives
reboots, the box, and even a model switch, and it keeps every run reproducible.

## 8. Freshness Gate (tiered)

Before generating, verify the underlying series genuinely re-parsed — using `run_logs` / parse
provenance, **not** the `as_of` date stamp (per the `cda414e` landmine: fresh as_of ≠ fresh parse).

- **Core series stale** → **skip.** Do not overwrite. Log `run_logs(source='briefing',
  status='skipped', reason='stale')`, alert. PWA keeps the last good briefing.
- **Only peripheral series stale** → **generate**, record the stale names in `stale_series`,
  and the PWA shows an honest banner naming them.
- **All fresh** → generate normally.

**Core vs peripheral (default; final list confirmed at plan time against `metric_definitions`):**
- **Core** (thesis-bearing): policy corridor (`policy_rate_repo/sdf/slf`), call money rate,
  FX reserves / import cover, key yield-curve tenors, CPI.
- **Peripheral:** everything else (individual fiscal lines, broad money, secondary commodities, etc.).

## 9. Display Rewiring — `Intelligence.tsx`

- **`useBriefing()` hook** (new, `src/hooks/`) — mirrors `useLiquidity.ts`: client-side PostgREST
  read of (a) the latest `briefings` row and (b) a lightweight history list (`week_of, title`).
- **Live wiring:** headline ← `title`; body ← `body`; anomaly sidebar ← `featured_anomalies`
  (real labels, real numbers, age from `generated_at`); both desktop + mobile layouts.
- **Removals:** Regenerate, Edit-tone (and Share unless wanted). Header "Updated …" becomes real
  (`generated_at`) or is removed.
- **History view:** "Read history" opens an in-page drawer/list reading the same table; selecting
  a week renders that stored briefing. No new route required.
- **Honesty:** `DemoBadge` null-gated (hides once a real row exists); a "data as of `data_as_of`"
  line + stale-series banner shown when `stale_series` is non-empty.
- **Types:** add a `Briefing` type to `src/lib/econdelta-metrics.ts` (or a new `src/lib/briefing.ts`).

## 10. Error Handling

Every failure path leaves the last good briefing untouched — the PWA never shows a half-built week.

| Situation | Behavior |
|---|---|
| Core series stale | Skip; `run_logs status='skipped' reason='stale'`; alert; no write |
| `run_max` fails (timeout / exit≠0 / non-JSON) | `MaxCallError`; `run_logs status='error'`; alert; no write |
| JSON fails schema (unknown `candidate_id`, missing field) | Reject; log; alert; no write |
| Re-run for same week | Idempotent `upsert` on `week_of` (same as `opus_review` same-day overwrite) |

## 11. Observability

Each run writes a `run_logs` row (`source='briefing'`) with `status`, `total_cost_usd`, tokens,
and duration (the telemetry `run_max` already returns), so cost/health is watchable alongside the
other timers (e.g. via `econdelta.clauding-lab.com/runs`).

## 12. Scheduling

- New systemd service + timer in `deploy/` (mirror existing `econdelta-*.service`/`.timer`),
  added to `install.sh`.
- `OnCalendar=Mon *-*-* 01:00:00` UTC = **07:00 BDT Monday** (box TZ confirmed at plan time;
  current units are UTC-based). Uses Sunday's 13:00 BDT aggregate; the freshness gate guarantees
  that cycle actually parsed.
- Optional retry timer (mirror the existing parse/aggregate retry pattern) if the first run skips on staleness.

## 13. Security & Honesty

- No new secrets: reuse `CLAUDE_CODE_OAUTH_TOKEN` (in `/etc/econdelta.env`) and the existing
  Supabase service-role key used by `utils/supabase_writer.py`.
- PWA reads via anon key only; `briefings` RLS is anon-SELECT / service-role-write.
- `--strict-mcp-config` + `--tools ""` keep the call hermetic (no MCP plugin routing, no tools).
- Honesty: tiered gate + `DemoBadge` gating + stale banner. A briefing is never published as
  "this week" on stale core data.

## 14. Testing Plan

Matches EconDelta's pytest layout + the `ECONDELTA_SKIP_OPUS_REVIEW` convention (no real Opus in CI):

- **Anomaly math** — table-driven RED/GREEN over fixture series (z-score, w/w Δ, threshold,
  per-frequency). Assert the *rule* (a known 2σ spike flags), not code shape.
- **Freshness gate** — stale-core / stale-peripheral / all-fresh fixtures → skip / banner / generate.
- **Schema validator** — accepts valid output; **rejects an unknown `candidate_id`** (the integrity guarantee).
- **`run_max` mocked** — subprocess patched (like existing tests); assert argv carries `--model opus[1m]`, `--effort xhigh`.
- **Idempotency** — same `week_of` upsert overwrites.
- **One gated live smoke test** — real Opus call behind an env flag, run manually, eyeball output.
- **YieldScope** — `useBriefing` hook test + render test (live row hides `DemoBadge`; empty →
  honest demo fallback); manual visual check at 320/768/1024/1440.
